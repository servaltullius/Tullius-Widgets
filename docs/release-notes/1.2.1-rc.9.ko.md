# Tullius Widgets v1.2.1-rc.9

## 변경 요약
- UI 다국어 처리를 외부 JSON 기반으로 전환했습니다. 이제 `PrismaUI/views/TulliusWidgets/i18n/manifest.json`과 언어별 `*.json` 파일만 추가하거나 수정하면 새 언어를 확장할 수 있습니다.
- 언어 선택 로직을 보강해 `manifest.json`의 `defaultLanguage`가 실제 기본 언어 결정에 반영되도록 했고, 게임 날짜/시간도 번역 파일의 월 이름과 포맷 패턴을 사용하도록 정리했습니다.
- WSL에서 작업 중일 때도 `./scripts/package.sh`, `scripts/release-local.ps1`, `scripts/verify-runtime-windows.ps1`가 Windows `xmake`와 `VsDevCmd.bat`를 자동으로 찾아 네이티브 빌드와 패키징을 수행하도록 릴리즈 경로를 안정화했습니다.
- 빌드/릴리즈 가이드와 에이전트 작업 지침을 최신 빌드 흐름에 맞게 문서화했습니다.

## 사용자 영향/호환성
- 기본 제공 언어는 한국어/영어 그대로지만, 이제 별도 DLL 재빌드 없이 외부 JSON 언어팩으로 다른 언어를 추가할 수 있습니다.
- 번역 파일 일부 키가 비어 있거나 로딩이 일시 실패해도 기본 번역으로 fallback하며, 초기 실패 후 재시도도 가능해졌습니다.
- 비한국어권 언어팩을 추가해도 Skyrim 날짜/시간 위젯이 영어 어순으로 고정되지 않고 번역 파일의 포맷을 따릅니다.
- 네이티브 DLL 자체 기능 호환성은 유지됩니다. 다만 이번 RC는 프런트 자산과 DLL을 함께 업데이트하는 것을 권장합니다.
- WSL 개발 환경에서는 직접 `xmake`를 호출하기보다 공식 스크립트를 사용하는 것이 권장 경로입니다.

## 설치/업데이트 안내
1. `v1.2.1-rc.9` 프리릴리즈 zip을 다운로드합니다.
2. 압축 안의 `SKSE/`와 `PrismaUI/` 폴더를 게임의 `Data/` 폴더에 복사하거나 모드 매니저로 그대로 설치합니다.
3. 기존 버전을 덮어쓴 뒤 게임을 완전히 종료하고 다시 실행해 DLL과 UI 자산이 함께 새로 로드되도록 합니다.
4. 다른 언어를 추가하려면 `Data/PrismaUI/views/TulliusWidgets/i18n/` 아래에 새 언어 JSON 파일을 넣고 `manifest.json`의 `languages` 항목을 함께 갱신합니다.
